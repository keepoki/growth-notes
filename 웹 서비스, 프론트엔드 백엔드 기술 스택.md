
## 프론트엔드

| **카테고리**       | **추천 기술**                                            | **선정 이유 및 한국 시장 특징**                             |
| -------------- | ---------------------------------------------------- | ------------------------------------------------ |
| **언어**         | **TypeScript**                                       | 선택이 아닌 **필수**                                    |
| **프레임워크**      | **Next.js (App Router)**                             | 신규 프로젝트는 대부분 App Router, 레거시는 Pages Router       |
| **서버 상태 관리**   | **TanStack Query (React Query)**                     | 서버 상태 관리는 무조건 TanStack Query                     |
| **클라이언트 상태**   | **Redux**, **Zustand**                               | 신규 프로젝트는 대부분 Zustand, 레거시는 Redux (Redux Toolkit) |
| **스타일링**       | **Styled-Components**, **Emotion**, **Tailwind CSS** | 신규 프로젝트는 대부분 TailwindCSS, 레거시는 CSS in JS 방식을 사용  |
| **유효성 검사**     | **React Hook Form** + **Zod**                        | 폼 관리는 RHF, 스키마 유효성 검사로 Zod를 가장 많이 사용             |
| **인증 (Auth)**  | **NextAuth.js (Auth.js)**                            | 소셜 로그인 연동 기능 등 간편 인증 가능해서 많이 사용                  |
| **문서화/협업**     | **Storybook**                                        | 필수는 아니나, 디자이너/기획자와의 협업을 중시하는 회사에서 많이 사용          |
| **패키지 매니저**    | **yarn berry** (또는 pnpm)                             | npm보다 yarn(v1)이나 yarn berry(v3+)를 선호하는 기업이 많음    |
| **날짜/시간**      | **Day.js**, **date-fns**                             | Day.js, date-fns가 표준. 레거시는 Moment.js             |
| **데이터 시각화**    | **Recharts** (또는 Chart.js)                           | 그래프/차트를 그릴 때 가장 많이 사용. React 친화적입니다.             |
| **에디터**        | **Toast UI Editor**, **React-Quill**                 | 게시판 에디터 가장 많이 사용. Toast UI Editor는 NHN이 만듦       |
| **애니메이션**      | **Motion**, **GSAP**                                 | 인터렉티브 애니메이션 라이브러리                                |
| **실시간 통신**     | **Socket.io-client**                                 | 백엔드와 채팅, 실시간 알림 기능을 구현할 때 씁니다.                   |
| **가상화 (List)** | **React-Virtual**                                    | 무한 스크롤 렌더링하는 최적화 라이브러리                           |
| **아이콘**        | **React Icons**                                      | 이 라이브러리 하나로 모든 아이콘(Material, FA 등)을 가져다 쓸 수 있음   |
| **SEO 향상**     | **next-seo**                                         | 검색엔진 최적화                                         |
| **다국어**        | **next-i18next**                                     | 제일 많이 사용                                         |


## 백엔드

| **카테고리**      | **추천 기술**                 | **선정 이유 및 한국 시장 특징**                                                                                    |
| ------------- | ------------------------- | ------------------------------------------------------------------------------------------------------- |
| **프레임워크**     | **NestJS**                | Java Spring을 선호하는 한국, 구조가 유사한 NestJS를 많이 사용                                                             |
| **데이터베이스**    | **PostgreSQL** (or MySQL) | 성능 뛰어남, 무료 라이선스                                                                                         |
| **ORM**       | **TypeORM** (or Prisma)   | **TypeORM**을 가장 많이 사용합니다. Prisma는 스타트업에서 선호                                                             |
| **인증 (Auth)** | **Passport.js** + **JWT** | NestJS의 Guard 시스템과 Passport 전략을 결합하여 구현하는 것이 정석                                                         |
| **유효성 검사**    | **class-validator**       | DTO(데이터 전송 객체)에 데코레이터를 붙여 검증하는 방식이 NestJS 표준                                                            |
| **API 문서화**   | **Swagger**               | API 명세에 가장 많이 쓰임                                                                                        |
| **로깅**        | **Winston**               | nest-winston을 사용하여 운영 환경(Prod)과 개발 환경(Dev) 로그를 분리해 저장하는 경험이 중요                                          |
| **캐싱 & 세션**   | **Redis** (필수)            | 1) 자주 조회하는 데이터 캐싱, 2) Refresh Token 저장, 3) 이메일 인증 코드 만료 관리 등에 사용합니다. RDB(DB) 부하를 줄이는 핵심 기술입니다.          |
| **검색 엔진**     | **Elasticsearch**         | 단순 DB LIKE 검색은 느림. 상품 검색, 자동 완성 기능을 구현할 때 사용                                                            |
| **메시지 큐**     | **BullMQ** (Redis 기반)     | **Kafka/RabbitMQ**는 인프라 비용이 큼. Node.js 환경에서는 **Redis를 이용한 BullMQ**로 이메일 발송, 알림 처리 등 비동기 작업을 처리하는 것이 효율적 |
| **대용량 큐**     | **Kafka**                 | 대용량 로그 처리가 필요한 곳에서 씀                                                                                    |
| **파일 업로드**    | **AWS S3** + **Multer**   | 이미지는 서버에 직접 저장하지 않고 AWS S3 같은 스토리지에 저장하는 것이 국룰입니다. (multer-s3 라이브러리 사용)                                 |
| **스케줄링**      | **@nestjs/schedule**      | 매일 밤 12시에 정산을 한다거나, 알림을 보내는 등의 배치(Batch) 작업에 사용                                                         |
